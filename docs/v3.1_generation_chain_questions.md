## Client feedback being addressed

---

## Proposed architecture (summary)

A 6-agent chain on the generation side (mirroring our retrieval-side tool fan-out):

1. **Memory Recall Agent** — pulls rolling summary + last 6 turns + top-3 semantically related older turns
2. **Query Rewriter Agent** — resolves coreference, makes query self-contained
3. **ReAct Retrieval Agent** — UNCHANGED (current `agent.py`)
4. **Answer Synthesizer Agent** — turns 30K-char drawing dumps into tight prose
5. **Style/Tone Rewriter Agent** — final humanization pass
6. **Memory Writer (background)** — writes turn + embedding + updates rolling summary

All agents flag-gated. All flags default OFF → behaviour identical to today's v3.
